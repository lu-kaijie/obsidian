# 自我改进学习循环 — 实现解析

## 概述

Hermes Agent 的自我改进学习循环是一个闭环系统，使 Agent 能够从每次对话中提取经验，
将其固化为可复用的"技能"（Skills），并在后续使用中持续优化这些技能。整个循环包含
四个核心阶段：**技能创建 → 使用追踪 → 后台审查 → 生命周期管理**。

核心源码文件：

| 文件 | 职责 |
|------|------|
| `tools/skill_manager_tool.py` | 技能 CRUD 工具（create/edit/patch/delete） |
| `tools/skill_usage.py` | 使用遥测与来源追踪 |
| `agent/curator.py` | 后台维护编排器（状态转换 + LLM 审查） |
| `run_agent.py` | Nudge 触发 + 后台审查 fork |

## 1. 技能创建机制

### 1.1 skill_manage 工具

Agent 通过 `skill_manage` 工具管理技能，支持以下操作：

```python
def skill_manage(
    action: str,      # create | edit | patch | delete | write_file | remove_file
    name: str,        # 技能名称
    content: str,     # SKILL.md 全文（create/edit 时）
    category: str,    # 分类目录
    file_path: str,   # 子文件路径（write_file/remove_file）
    old_string: str,  # patch 时的查找文本
    new_string: str,  # patch 时的替换文本
)
```

技能存储在 `~/.hermes/skills/<category>/<name>/SKILL.md`，支持附带：
- `references/` — 参考资料、API 文档摘录
- `templates/` — 可复用的模板文件
- `scripts/` — 辅助脚本

### 1.2 技能的目标形态

系统追求的是**类级别（class-level）技能**，而非一次会话一个技能的碎片化条目。
一个好的技能应该是一个"伞形"知识节点，包含丰富的 SKILL.md 正文和子目录支撑文件。

## 2. Nudge 触发机制

### 2.1 迭代计数器

在 `run_agent.py` 的 `AIAgent.__init__` 中：

```python
self._skill_nudge_interval = 10  # 默认每 10 次工具迭代触发一次
skills_config = config.get("skills", {})
self._skill_nudge_interval = int(
    skills_config.get("creation_nudge_interval", 10)
)
```

每次对话结束时检查：

```python
if (self._skill_nudge_interval > 0
        and self._iters_since_skill >= self._skill_nudge_interval
        and "skill_manage" in self.valid_tool_names):
    _should_review_skills = True
    self._iters_since_skill = 0
```

### 2.2 触发条件

当一次对话的工具调用迭代次数达到阈值（默认 10 次），说明这是一次
"有实质工作量"的对话，值得从中提取经验。此时系统标记需要进行技能审查。

## 3. 后台审查 Agent

### 3.1 Fork 机制

`_spawn_background_review()` 在对话响应交付给用户**之后**运行，
不与用户任务竞争模型注意力：

```python
def _spawn_background_review(self, messages_snapshot, review_memory, review_skills):
    # 选择 prompt
    if review_memory and review_skills:
        prompt = self._COMBINED_REVIEW_PROMPT
    elif review_memory:
        prompt = self._MEMORY_REVIEW_PROMPT
    else:
        prompt = self._SKILL_REVIEW_PROMPT

    # 在后台线程中 fork 一个 AIAgent
    review_agent = AIAgent(
        model=self.model,
        max_iterations=16,
        quiet_mode=True,
        # 继承父 agent 的凭证、provider、缓存的系统提示
    )
```

关键设计：
- 继承父 agent 的 `_cached_system_prompt`，命中前缀缓存（节省 ~26% 成本）
- 工具白名单限制为 `memory` + `skills` 工具集
- 自动拒绝危险命令（auto-deny approval callback）
- 静默运行，不产生用户可见输出

### 3.2 技能审查 Prompt

`_SKILL_REVIEW_PROMPT` 指导审查 agent 寻找以下信号：

1. **用户纠正了风格/格式/冗余度** — 更新相关技能嵌入偏好
2. **用户纠正了工作流/方法/步骤顺序** — 编码为技能中的陷阱或步骤
3. **出现了非平凡的技术/修复/调试路径** — 捕获为未来会话的参考
4. **已加载的技能被证明有误或过时** — 立即 patch

操作优先级：
1. 更新当前已加载的技能（最高优先）
2. 更新已有的伞形技能（通过 skills_list + skill_view 查找）
3. 在已有技能下添加支撑文件（references/templates/scripts）
4. 创建新技能（仅当无现有技能覆盖该领域时）

## 4. 使用遥测（skill_usage.py）

### 4.1 数据存储

遥测数据存储在 `~/.hermes/skills/.usage.json`，按技能名索引：

```json
{
  "my-skill": {
    "use_count": 5,
    "view_count": 12,
    "patch_count": 3,
    "last_used_at": "2026-05-10T08:30:00+00:00",
    "last_viewed_at": "2026-05-15T14:00:00+00:00",
    "last_patched_at": "2026-05-12T09:00:00+00:00",
    "created_at": "2026-04-01T10:00:00+00:00",
    "state": "active",
    "pinned": false
  }
}
```

### 4.2 来源过滤

Curator 只管理 **agent 创建的技能**，通过排除以下来源实现：
- `.bundled_manifest` 中列出的内置技能
- `.hub/lock.json` 中记录的 Hub 安装技能
- 手动创建的技能不会被推断为 agent 创建

## 5. Curator 生命周期管理

### 5.1 状态机

```
active ──(闲置 > stale_after_days)──→ stale
stale  ──(闲置 > archive_after_days)──→ archived
stale  ──(被使用)──→ active（自动重新激活）
```

默认配置：
- `stale_after_days`: 30 天无活动 → 标记为 stale
- `archive_after_days`: 90 天无活动 → 归档到 `.archive/`

### 5.2 自动状态转换

`apply_automatic_transitions()` 是纯函数（无 LLM），遍历所有 agent 创建的技能：

```python
for row in agent_created_report():
    if row.get("pinned"):
        continue  # pinned 技能永不自动转换
    anchor = last_activity or created_at
    if anchor <= archive_cutoff:
        archive_skill(name)
    elif anchor <= stale_cutoff and state == "active":
        set_state(name, "stale")
    elif anchor > stale_cutoff and state == "stale":
        set_state(name, "active")  # 重新激活
```

### 5.3 LLM 审查 Pass

除了自动状态转换，Curator 还会 fork 一个 AIAgent 执行深度审查：
- 合并重复/重叠的技能为伞形技能
- 更新过时的技能内容
- 归档确实无用的技能

审查 agent 使用 `CURATOR_REVIEW_PROMPT`，强制遵守：
- 不触碰内置/Hub 安装的技能
- 不删除任何技能（最大破坏性操作是归档）
- 不触碰 pinned 技能

### 5.4 调度策略

Curator 不使用 cron 守护进程，而是**惰性触发**：

```python
def should_run_now():
    if not is_enabled(): return False
    if is_paused(): return False
    # 首次运行：种子化 last_run_at，延迟一个完整周期
    if last_run_at is None:
        state["last_run_at"] = now
        return False
    # 距上次运行超过 interval_hours（默认 7 天）
    return (now - last_run_at) >= interval_hours
```

额外门控：`min_idle_hours`（默认 2 小时）确保 agent 空闲时才运行。

### 5.5 安全保障

| 不变量 | 说明 |
|--------|------|
| 只管理 agent 创建的技能 | 内置/Hub 技能不受影响 |
| 永不删除 | 最大操作是归档到 `.archive/` |
| Pinned 豁免 | pinned 技能跳过所有自动转换和 LLM 审查 |
| 备份 | `curator_backup.py` 在每次运行前创建 tar.gz 快照 |
| Dry-run 模式 | `hermes curator run --dry-run` 只生成报告不执行 |

## 6. 完整循环图

```
用户对话（工具迭代 ≥ 10 次）
       │
       ▼
  触发 Skill Nudge
       │
       ▼
  Fork 后台审查 Agent ──→ 分析对话 ──→ 创建/更新技能
       │                                    │
       ▼                                    ▼
  skill_usage 记录遥测              技能库持续增长
       │
       ▼
  Curator 定期检查（7天周期）
       │
       ├─→ 自动状态转换（active/stale/archived）
       └─→ LLM 审查 Pass（合并/优化/归档）
```

这形成了一个完整的自我改进闭环：对话产生经验 → 经验固化为技能 →
技能在使用中被追踪 → 低价值技能被自动清理 → 高价值技能被持续优化。
