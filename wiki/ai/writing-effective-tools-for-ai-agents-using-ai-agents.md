# Writing effective tools for AI agents—using AI agents

**来源：** [Anthropic Engineering](https://www.anthropic.com/engineering/writing-tools-for-agents)

## 核心结论

这篇文章最值得保留的，是它把“怎么写工具”从 schema 设计拉高到完整方法论：工具定义、原型、评测、与 agent 协作地迭代工具本身。也就是说，工具不是静态配件，而是需要被持续设计和验证的接口。

## 为什么值得看

- 它是工具设计的一手工程实践页。
- 和 [[wiki/ai/function-calling-mcp-skills-differences-practice|Function Calling、MCP 与 Skills 的边界]]、[[wiki/ai/code-execution-with-mcp-building-more-efficient-ai-agents|Code execution with MCP: building more efficient AI agents]] 构成工具层三件套。
- 也能帮助理解为什么工具描述质量经常比模型能力更影响 agent 成败。
